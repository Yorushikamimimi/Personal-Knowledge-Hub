# 构建与运行

## 文档说明

本文档用于本地验证当前工程是否具备最小可运行能力，并确认构建产物可以正常启动。

适用版本：`V0.4-beta`

## 环境前提

- Node.js 已安装
- npm 可用
- 项目依赖已在根目录安装
- 当前工程不依赖数据库和外部 CMS

## 验证命令总览

| 步骤 | 命令 | 目的 | 预期结果 |
| --- | --- | --- | --- |
| 1 | `npm install` | 安装依赖 | 无报错完成安装 |
| 2 | `npm run build` | 执行生产构建 | 构建成功，无阻塞错误 |
| 3 | `npm run start` | 启动生产服务 | 可本地访问站点 |
| 4 | `npm run dev` | 启动开发模式 | 可本地访问站点 |

## 本地运行步骤

### 1. 安装依赖

```bash
npm install
```

检查项：
- [ ] 依赖安装完成。
- [ ] 未出现关键依赖缺失。
- [ ] 未出现锁文件冲突导致的中断。

### 2. 生产构建验证

```bash
npm run build
```

检查项：
- [ ] 构建成功。
- [ ] 已识别首页、Notes、Downloads、Topics、About、合规页路由。
- [ ] published 的动态详情页生成成功。
- [ ] draft 内容未进入静态构建页面。

### 3. 生产启动验证

```bash
npm run start
```

检查项：
- [ ] 服务启动成功。
- [ ] 浏览器可访问首页。
- [ ] 主要页面可正常跳转。
- [ ] 详情页可正常打开。

### 4. 开发模式验证

```bash
npm run dev
```

检查项：
- [ ] 开发服务启动成功。
- [ ] 页面访问无白屏。
- [ ] 控制台无明显阻塞错误。
- [ ] 本地修改文档或内容后可重新加载。
