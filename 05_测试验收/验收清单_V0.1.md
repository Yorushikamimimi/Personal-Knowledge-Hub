# 阶段 4 测试验收总入口

## 说明

此文档用于统一收口 `Personal Knowledge Hub` 当前 `V0.4-beta` 版本的阶段 4 测试验收资料，帮助个人或团队按固定顺序执行本地验收。

适用范围：
- 首页精选区
- Notes 列表页与详情页
- Downloads 列表页与详情页
- Topics 聚合页
- About 与合规页
- layout / Header / Footer / PageContainer
- draft 内容过滤验证

## 使用方法

- 建议先查看验收总览，了解本轮范围和目标。
- 再按“构建与运行 -> 功能验证 -> 样式与布局 -> draft 内容验证”的顺序执行。
- 每份文档都可以独立勾选、补充结果和备注。
- 若发现问题，建议同步记录到项目的开发执行或 Bug 文档中。

## 1. 验收总览

- [验收总览表.md](./验收总览表.md)：汇总阶段 4 所有验收模块、页面范围、重点检查项和最终结论。

## 2. 构建与运行

- [构建与运行.md](./构建与运行.md)：用于执行 `npm install`、`npm run build`、`npm run start`、`npm run dev` 的本地验证步骤。

## 3. 功能验证

- [功能验证.md](./功能验证.md)：用于检查首页精选区、Notes、Downloads、Topics、合规页的访问、跳转、按钮和详情展示链路。

## 4. 样式与布局

- [样式与布局.md](./样式与布局.md)：用于检查 PageContainer、导航、页脚、卡片、双列 / 三列展示和详情页排版的一致性。

## 5. draft 内容验证

- [draft 内容验证.md](./draft 内容验证.md)：用于确认 draft Note、draft Download 和 draft Topic 不出现在列表页、首页精选区和公开详情页。

## 推荐执行顺序

1. 先执行 [验收总览表.md](./验收总览表.md)，明确本轮范围。
2. 再执行 [构建与运行.md](./构建与运行.md)，确认工程可运行、可构建。
3. 然后执行 [功能验证.md](./功能验证.md)，完成页面与跳转链路验收。
4. 接着执行 [样式与布局.md](./样式与布局.md)，确认结构和展示一致性。
5. 最后执行 [draft 内容验证.md](./draft 内容验证.md)，确认公开内容过滤规则生效。
