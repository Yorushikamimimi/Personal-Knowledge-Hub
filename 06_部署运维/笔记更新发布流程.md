# 笔记更新发布流程（固定单一路径）

## 结论

笔记发布与功能发布统一，全部走 **Bundle 离线发布**。

## 适用范围

- `content/notes/**/*.mdx` 新增或修改
- 以及任何首页/样式/脚本改动

## 前置要求

- 新笔记 frontmatter 至少包含：
  - `slug`
  - `status: "published"`
- 本地已通过：`npm run build`

## 唯一发布流程

### 1) 本地提交并推送（用于远程备份与历史记录）

```bash
cd "D:/Workspace/Personal Knowledge Hub"
git add -A
git commit -m "content(notes): update notes"
git push origin main
```

### 2) 本地打包并上传 Bundle

```bash
cd "D:/Workspace/Personal Knowledge Hub"
$ts = Get-Date -Format yyyyMMdd-HHmmss
$bundle = "pkh-main-$ts.bundle"
git bundle create $bundle main
scp $bundle root@81.68.72.245:/srv/sites/personal-knowledge-hub/app/
```

### 3) 服务器部署（固定入口）

```bash
cd /srv/sites/personal-knowledge-hub/app
bash scripts/ops/deploy-bundle.sh ./pkh-main-<timestamp>.bundle main
```

## 验收

```bash
curl -I https://yoruming.cn
curl -s "https://yoruming.cn/notes?t=$(date +%s)" | grep -E "当前已发布|<slug>"
```

## 禁用项

- 不再使用 `deploy-notes.sh online/offline` 作为日常发布入口
- 不再使用 `scp` 单文件直传到 `content/notes` 后直接 build

## 术语说明

`Bundle`：Git 离线仓库包，避免服务器访问 GitHub 失败。
