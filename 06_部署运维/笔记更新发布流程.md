# 笔记更新发布流程

## 1. 目标

把新笔记从本地发布到线上，避免每次手工排查。

## 2. 适用范围

- 新增/修改 `content/notes/**/*.mdx`
- 不改后端逻辑，仅内容发布

## 3. 前置要求

- 本地已完成 `git push origin main`
- 新笔记 frontmatter 含：
  - `slug`（URL 标识）
  - `status: "published"`（发布状态）

## 4. 标准操作（推荐）

### 4.1 本地操作

```bash
cd "D:/Workspace/Personal Knowledge Hub"
npm run build
git add -A
git commit -m "content(notes): add <slug>"
git push origin main
```

### 4.2 服务器操作（网络正常）

```bash
cd /srv/sites/personal-knowledge-hub/app
bash scripts/ops/deploy-notes.sh online
```

## 5. 应急操作（服务器拉 GitHub 超时）

### 5.1 本地把笔记文件直接传到服务器

```bash
scp "D:/Workspace/Personal Knowledge Hub/content/notes/interview/<file>.mdx" root@81.68.72.245:/srv/sites/personal-knowledge-hub/app/content/notes/interview/
```

### 5.2 服务器离线发布（不拉 Git）

```bash
cd /srv/sites/personal-knowledge-hub/app
bash scripts/ops/deploy-notes.sh offline
```

## 6. 发布后验收

```bash
curl -s "https://yoruming.cn/notes?t=$(date +%s)" | grep -E "<slug>|当前已发布"
```

可选：按标题关键字校验

```bash
VERIFY_PATTERN='codex-gpt-sop|md-github|prompt-ai' bash scripts/ops/deploy-notes.sh offline
```

## 7. 常见问题

1. 页面没看到新笔记：先确认服务器目录里是否存在新 `.mdx` 文件。
2. 文件存在但页面不更新：执行 `bash scripts/ops/deploy-notes.sh offline`。
3. 出现 client-side exception：先无痕访问并强刷；再看浏览器 Console 首条红色报错。
