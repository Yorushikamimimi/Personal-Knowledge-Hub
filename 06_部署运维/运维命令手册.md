# 运维命令手册（固定单一路径）

## 结论

从 2026-03-24 起，线上发布只使用一条方法：

- `Bundle`（离线仓库包）上传
- 服务器执行：`bash scripts/ops/deploy-bundle.sh <bundle-file> main`

不再把 `git pull/fetch origin` 作为发布 SOP（标准操作流程）。

## 发布前检查

```bash
cd "D:/Workspace/Personal Knowledge Hub"
git status -sb
npm run build
```

要求：本地构建通过后再打包发布。

## 标准发布流程（唯一）

### 1) 本地打包

```bash
cd "D:/Workspace/Personal Knowledge Hub"
$ts = Get-Date -Format yyyyMMdd-HHmmss
$bundle = "pkh-main-$ts.bundle"
git bundle create $bundle main
scp $bundle root@81.68.72.245:/srv/sites/personal-knowledge-hub/app/
```

### 2) 服务器部署

```bash
cd /srv/sites/personal-knowledge-hub/app
bash scripts/ops/deploy-bundle.sh ./pkh-main-<timestamp>.bundle main
```

## 发布后验收（固定 5 项）

```bash
pm2 status
ss -lntp | grep :3001
curl -I http://127.0.0.1:3001
curl -I https://yoruming.cn
curl -I https://music.yoruming.cn
```

## 故障排查（最小命令）

```bash
pm2 logs pkh-web --lines 200
```

## 禁用发布方式（避免分叉）

以下命令不再作为发布方法：

- `git pull --ff-only origin main`
- `git fetch origin main`
- 服务器手工编辑业务代码并直接 build
- 运行中删除 `.next` 后不重启 PM2

## 说明

`Bundle` 是 Git 的离线仓库包，适合服务器外网不稳定场景；
`SOP` 是标准操作流程，表示后续只按这一条执行。
