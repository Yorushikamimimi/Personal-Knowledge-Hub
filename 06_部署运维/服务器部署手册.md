# 服务器部署手册

## 1. 目标

在腾讯云 Ubuntu 22.04 服务器上，把站点统一部署到一个目录，使用 `Nginx + PM2` 管理，且不占用新的公网端口。

从 2026-03-24 起，发布路径固定为 **Bundle 离线发布**（单一路径）。

## 2. 统一目录规范

统一使用一个根目录：

```text
/srv/sites/personal-knowledge-hub
├─ app/       # Git 代码、node_modules、.next
├─ logs/      # 站点运行日志（预留）
└─ backups/   # 回滚标记
```

## 3. 端口策略（强约束）

- 站点应用仅监听：`127.0.0.1:3001`
- 对外流量只走现有 `Nginx` 的 `80/443`
- 不新增公网开放端口

## 4. 首次部署步骤

### 4.1 安装运行时与初始化目录

在服务器中进入项目仓库后执行：

```bash
bash scripts/ops/bootstrap-ubuntu-22.04.sh
```

### 4.2 通过 Bundle 离线上线（固定方法）

本地执行：

```bash
cd "D:/Workspace/Personal Knowledge Hub"
$ts = Get-Date -Format yyyyMMdd-HHmmss
$bundle = "pkh-main-$ts.bundle"
git bundle create $bundle main
scp $bundle root@81.68.72.245:/srv/sites/personal-knowledge-hub/app/
```

服务器执行：

```bash
cd /srv/sites/personal-knowledge-hub/app
bash scripts/ops/deploy-bundle.sh ./pkh-main-<timestamp>.bundle main
```

### 4.3 接入 Nginx

1. 复制示例配置：

```bash
sudo cp scripts/ops/nginx-site.conf.example /etc/nginx/conf.d/pkh-web.conf
```

2. 按实际域名调整 `server_name`。  
3. 检查并重载：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

### 4.4 验证

```bash
bash scripts/ops/healthcheck.sh yoruming.cn
```

## 5. 日常更新发布（唯一 SOP）

每次更新都执行同一条路径：

1) 本地：提交、推送、打包、上传

```bash
cd "D:/Workspace/Personal Knowledge Hub"
npm run build
git add -A
git commit -m "feat: <change>"
git push origin main

$ts = Get-Date -Format yyyyMMdd-HHmmss
$bundle = "pkh-main-$ts.bundle"
git bundle create $bundle main
scp $bundle root@81.68.72.245:/srv/sites/personal-knowledge-hub/app/
```

2) 服务器：部署并验收

```bash
cd /srv/sites/personal-knowledge-hub/app
bash scripts/ops/deploy-bundle.sh ./pkh-main-<timestamp>.bundle main
```

## 6. 故障回滚

查看提交：

```bash
cd /srv/sites/personal-knowledge-hub/app
git log --oneline -n 10
```

回滚指定提交：

```bash
bash scripts/ops/rollback.sh <commit-sha>
```

## 7. 接入 music 二级域名

当 DNS 已有 `music -> 服务器公网 IP` 后，执行：

```bash
cd /srv/sites/personal-knowledge-hub/app

sudo cp scripts/ops/nginx-music-site.conf.example /etc/nginx/conf.d/music-web.conf
sudo nginx -t
sudo systemctl reload nginx

sudo certbot --nginx -d music.yoruming.cn --redirect
```

验证：

```bash
curl -I http://music.yoruming.cn
curl -I https://music.yoruming.cn
```

## 8. 禁用项

以下方式不再作为上线 SOP：

- `git fetch origin` / `git pull --ff-only origin main`
- 服务器手工改业务代码后直接 build
- 单文件 `scp` 到 `content` 后直接发布
