# 服务器部署手册

## 1. 目标

在腾讯云 Ubuntu 22.04 服务器上，把站点统一部署到一个目录，使用 `Nginx + PM2` 管理，且不占用新的公网端口。

## 2. 统一目录规范

统一使用一个根目录：

```text
/srv/sites/personal-knowledge-hub
├─ app/       # Git 代码、node_modules、.next
├─ logs/      # 站点运行日志（预留）
└─ backups/   # 回滚标记
```

## 3. 端口策略（强约束）

- 站点应用仅监听：`127.0.0.1:3001`
- 对外流量只走现有 `Nginx` 的 `80/443`
- 不新增公网开放端口
- 不碰已占用的宿主机关键端口（80/81/443/9090/8068/8443/3306/5432/6379/8848/9848/9849/1935/8080/5003/23333/24444/22/53）

## 4. 首次部署步骤

### 4.1 安装运行时与初始化目录

在服务器中进入项目仓库后执行：

```bash
bash scripts/ops/bootstrap-ubuntu-22.04.sh
```

### 4.2 拉取并构建上线

```bash
bash scripts/ops/deploy.sh
```

### 4.3 接入 Nginx

1. 复制示例配置：

```bash
sudo cp scripts/ops/nginx-site.conf.example /etc/nginx/conf.d/pkh-web.conf
```

2. 按实际域名调整 `server_name`。  
3. 检查并重载：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

### 4.4 验证

```bash
bash scripts/ops/healthcheck.sh yoruming.cn
```

## 5. 日常更新发布

每次本地 `push` 到 `main` 后，在服务器执行：

```bash
cd /srv/sites/personal-knowledge-hub/app
git fetch origin main
git pull --ff-only origin main
bash scripts/ops/deploy.sh
```

## 6. 故障回滚

查看提交：

```bash
cd /srv/sites/personal-knowledge-hub/app
git log --oneline -n 10
```

回滚指定提交：

```bash
bash scripts/ops/rollback.sh <commit-sha>
```
