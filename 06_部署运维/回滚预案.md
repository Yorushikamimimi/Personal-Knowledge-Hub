# 回滚预案

## 1. 触发条件

满足任一条件即触发回滚：

- 发布后首页或主路径持续 `5xx`
- 发布后核心页面白屏或无法访问
- 发布后 Nginx 正常但应用健康检查失败

## 2. 回滚目标

在 5~10 分钟内，把线上恢复到“上一个可用提交”。

## 3. 回滚步骤

### 3.1 查可用提交

```bash
cd /srv/sites/personal-knowledge-hub/app
git log --oneline -n 10
```

### 3.2 执行回滚

```bash
bash scripts/ops/rollback.sh <commit-sha>
```

脚本会自动执行：

1. checkout 目标提交
2. `npm ci`
3. `npm run build`
4. `pm2 reload`
5. 健康检查

### 3.3 验证

```bash
bash scripts/ops/healthcheck.sh yoruming.cn
```

## 4. 兜底操作

如果自动回滚脚本失败，使用手工兜底：

```bash
cd /srv/sites/personal-knowledge-hub/app
git checkout <commit-sha>
npm ci
npm run build
pm2 reload ecosystem.config.cjs --update-env
curl -I http://127.0.0.1:3001/
```

## 5. 回滚后动作

- 立刻记录故障窗口和提交号
- 暂停继续发布
- 用同一环境复现问题后再修复
