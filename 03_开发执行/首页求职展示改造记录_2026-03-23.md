# 首页求职展示改造记录（2026-03-23）

## 1. 改造目标

把首页从“知识站入口”调整为“后端 / 全栈项目作品集入口”，让招聘方和技术面试官优先看到项目能力。

## 2. 本轮改动（已落地）

1. Hero 区重写为求职展示文案：
   - 明确定位：Backend / Full-stack Portfolio
   - 明确主线：业务交付、数据一致性、可上线运维
   - 按钮聚焦：Projects / 技术型 Notes / GitHub

2. 首页信息架构调整为 Projects-first：
   - `Featured Projects` 保持前置并强化
   - `Latest Notes` 改为 `Project Notes`，作为支撑材料弱化展示
   - `Topics / Downloads / External Links` 保持在后部辅助区

3. Featured Projects 文案统一（招聘视角）：
   - `rag-nexus`：RAG、多源注入、pgvector、SSE
   - `nautilus-media-cloud`：异步任务调度、Worker、SKIP LOCKED
   - `nautilus-clinic`：门诊流程、RBAC、库存一致性、私有化部署

4. 首页 Notes 改为技术优先排序：
   - 对后端技术关键词加权
   - 对 workflow/prompt/SOP 类内容降权
   - 首页只取技术相关度更高的前 3 篇

5. 统计数字卡片替换为能力卡：
   - RAG Retrieval Chain
   - Async Scheduling
   - Business Workflow
   - Deploy Baseline

## 3. 代码文件

- `app/page.tsx`（首页结构、文案、排序策略）

## 4. 维护规则（后续内容更新）

1. 新增项目时：
   - 在 `projectCopy` 中补齐 `elevator + highlight` 两句文案，保持首页风格统一。

2. 新增笔记时：
   - 优先发布与项目强相关的技术总结（数据库、调度、架构、部署）。
   - workflow/prompt 模板类内容不建议占据首页主要位置。

3. 若后续需要加“简历导向”模块：
   - 建议作为二级区域补充，不覆盖 Projects 主线。

## 5. 验收建议

- 首页首屏是否在 5 秒内说明“你做过什么项目”。
- 面试官是否能在 1 次滚动内进入任一主打项目详情。
- Notes 是否从“主展示位”降为“证据补充位”。
