# 部署方案

## 1. 文档目标

本文档用于定义 Personal Knowledge Hub V0.1 从本地开发到构建，再到腾讯云服务器部署的基础链路与运行方式。

## 2. 部署边界

V0.1 部署方案遵循以下边界：

- 部署对象为单个 Next.js 前台站点
- 不引入数据库
- 不引入后台服务
- 不引入对象存储
- 不引入容器编排
- 运行方式以 Nginx + PM2 为主

## 3. 基础环境建议

### 3.1 本地开发环境

- Node.js LTS 版本
- npm 作为包管理器
- Windows 本地开发可行

### 3.2 服务器环境

- 腾讯云大陆服务器
- Linux 发行版建议使用 Ubuntu LTS
- Node.js LTS 版本
- Nginx
- PM2

## 4. 本地开发 -> build -> 腾讯云部署链路

### 4.1 本地开发阶段

1. 在本地维护代码、MDX 内容和静态资源
2. 本地完成页面验证与内容检查
3. 确认 `content`、`public`、应用代码结构可正常参与构建

### 4.2 本地构建阶段

1. 安装依赖
2. 执行生产构建
3. 验证构建产物可正常启动
4. 确认下载文件路径在生产构建后仍可访问

### 4.3 服务器部署阶段

1. 将代码与静态资源同步到腾讯云服务器
2. 在服务器安装依赖或使用已构建可部署产物
3. 执行生产构建
4. 通过 PM2 启动 Next.js 生产服务
5. 通过 Nginx 反向代理对外提供访问

### 4.4 发布校验阶段

1. 检查首页与所有主路径可访问
2. 检查 Note 与 DownloadResource 详情页路由可访问
3. 检查 `public/images` 与 `public/files/downloads` 中资源可访问
4. 检查下载链路是否为“列表页 -> 详情页 -> 下载”
5. 检查 About 与合规页是否可访问

## 5. 推荐运行方式：Nginx + PM2

### 5.1 PM2 职责

- 管理 Next.js 生产进程
- 负责进程守护、重启和开机自启
- 统一进程日志管理

### 5.2 Nginx 职责

- 对外暴露 80 / 443 端口
- 反向代理到 Next.js 生产端口，例如 `127.0.0.1:3000`
- 处理域名访问和 HTTPS 证书接入
- 为静态资源访问提供稳定入口

### 5.3 推荐链路

```text
用户浏览器
-> Nginx 80/443
-> Next.js Node 服务 127.0.0.1:3000
-> 页面与 public 静态资源响应
```

## 6. 目录处理建议

### 6.1 应用目录

建议将应用部署在独立目录，例如：

```text
/var/www/personal-knowledge-hub
```

### 6.2 静态资源目录

以下资源随项目一并部署：

- `public/images/*`
- `public/icons/*`
- `public/files/downloads/*`

### 6.3 下载文件目录建议

V0.1 推荐将下载文件与应用一同部署，统一纳入项目版本内：

```text
public/files/downloads/*
```

原因：

1. 简化部署流程
2. 保持 metadata 与实体文件同步
3. 不额外引入对象存储和外部文件服务

说明：如果未来下载文件体积明显增大，可再考虑将下载目录迁移到独立存储，但不属于 V0.1 范围。

## 7. 发布方式建议

### 7.1 推荐方式

- 代码仓库拉取到服务器
- 服务器执行安装与构建
- 使用 PM2 启动生产服务

### 7.2 版本更新建议

1. 每次发布都应包含代码、`content` 和 `public` 的完整变更
2. 更新下载资源时，应同时更新：
   - `content/downloads/*.mdx`
   - `public/files/downloads/*`
3. 发布前应校验 `downloadPath` 对应文件是否存在

## 8. 环境变量建议

V0.1 建议仅保留最少环境变量，例如：

- `NODE_ENV=production`
- `PORT=3000`
- `SITE_URL=https://your-domain.com`

说明：V0.1 不需要数据库连接、鉴权密钥、对象存储密钥等复杂环境变量。

## 9. Nginx 与 PM2 的运行建议

### 9.1 PM2 建议

- 使用单应用名称管理站点进程
- 开启开机自启
- 日志按标准输出和错误输出区分

### 9.2 Nginx 建议

- 主域名统一转发到 Next.js 服务
- 保持对 `/_next/static/*`、`/images/*`、`/files/downloads/*` 等静态资源路径的稳定访问
- HTTPS 证书接入放在 Nginx 层处理

## 10. 部署校验清单

上线前至少检查：

1. `npm run build` 可成功完成
2. PM2 进程可正常启动并稳定运行
3. Nginx 反向代理无 502 问题
4. 所有主页面可访问
5. 下载资源详情页可访问
6. 下载按钮指向的文件可正常下载
7. About 与合规页 metadata 和正文可正常显示

## 11. V0.1 部署结论

V0.1 推荐采用“本地开发 -> 本地验证 -> 服务器构建 -> PM2 启动 -> Nginx 反向代理”的基础部署方案。该方案足够支撑当前内容型站点，并且与“无数据库、无后台、静态资源随项目部署”的技术边界保持一致。
